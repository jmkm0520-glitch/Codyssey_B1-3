## 프로젝트 1. 자동화 도구 비교 구현 (Make vs Zapier)

### 1. 프로젝트 개요
본 프로젝트는 동일한 업무 자동화 워크플로우를 Make와 Zapier 두 가지 노코드 자동화 플랫폼으로 구현하고 비교 분석하는 것을 목표로 하였다. 자동화 대상 업무는 교수자의 학생 성적 관리 업무로 선정하였다. 학생 성적이 Google Sheets에 입력되면 학점을 기준으로 학사경고 여부를 판단하고, 이메일을 자동 발송하며 결과를 별도 시트에 저장하는 자동화 시스템을 구축하였다.

### 2. 자동화 대상 업무
**업무 설명**: 교수자는 학생들의 성적을 확인하고 학사경고 여부를 판단한 뒤 개별적으로 결과를 통보해야 한다. 기존에는 다음과 같은 수작업이 필요하였다.

- 학생 성적 확인
- 학사경고 여부 판단
- 이메일 발송
- 결과 정리

이를 자동화하여 반복 업무를 최소화하고자 하였다.

### 3. 워크플로우 설계
**자동화 흐름**
```
Google Sheets
↓ 학점 확인
↓ 조건 분기
├─ 학점 = F
│   ↓ 학사경고 메일 발송
│   ↓ 학사경고 안내 시트 저장
└─ 학점 ≠ F
    ↓ 성적 결과 메일 발송
    ↓ 성적결과 시트 저장
```

### 4. Make 구현
**사용 모듈**: Google Sheets - Watch New Rows, Router, Gmail - Send Email, Google Sheets - Add a Row

**워크플로우**
```
Google Sheets
↓ Router
├─ 학점 = F
│   ↓ Gmail
│   ↓ Google Sheets
└─ 학점 ≠ F
    ↓ Gmail
    ↓ Google Sheets
```

  2. 일반 학생 (학점 ≠ F)
- [캡처 4] Gmail 설정 화면
  1. 학사경고
  2. 성적 결과
- [캡처 5] Google Sheets 저장 설정

### 8. 문제 발생 및 해결 과정
프로젝트 구현 과정에서 여러 오류가 발생하였다.

**문제 1. Gmail 발송 오류**
- 증상: Gmail 모듈 실행 시 이메일 발송 실패
- 원인: Google Sheets에 입력된 이메일 주소 형식이 올바르지 않았음
- 해결: 이메일 형식을 수정하고 Gmail 모듈의 매핑 정보를 재설정하여 해결하였다.

**문제 2. Google Sheets 데이터 매핑 오류**
- 증상: 이름, 평균, 학점 값이 잘못 저장되거나 비어 있는 상태로 저장됨
- 원인: Google Sheets 컬럼과 Make 모듈의 매핑이 올바르게 연결되지 않았음
- 해결: 각 컬럼을 다시 매핑하고 테스트 데이터를 이용하여 정상 저장 여부를 확인하였다.

**문제 3. Trigger 미작동**
- 증상: 새로운 행을 추가해도 자동화가 실행되지 않음
- 원인: Trigger 시작 위치와 실행 주기 설정 문제
- 해결: Trigger 시작 위치를 재설정하고 Run once 테스트를 반복하여 정상 동작을 확인하였다.

**문제 4. 시트 저장 실패**
- 증상: 일부 데이터가 결과 시트에 저장되지 않음
- 원인: Google Sheets Add Row 모듈의 시트 선택 및 컬럼 매핑 오류
- 해결: 저장 대상 시트와 컬럼 정보를 다시 연결하여 문제를 해결하였다.

**결과**

오류 수정 후 다음을 확인하였다.
- Gmail 자동 발송 성공
- 학사경고 시트 저장 성공
- 성적결과 시트 저장 성공

**동일한 요구를 여러 도구로 시도했을 때 결과 차이**

본 프로젝트에서는 동일한 자동화 업무(성적 자동 계산, 학사경고 분기, 이메일 발송, 결과 저장)를 Make와 Zapier에서 각각 구현하였다. 구현 결과 두 도구 모두 동일한 자동화 기능을 수행할 수 있었으나 구현 방식과 사용 경험에는 차이가 있었다.

Make는 Router를 이용하여 조건 분기를 시각적으로 표현할 수 있어 데이터 흐름을 한눈에 파악할 수 있었다. 또한 각 모듈의 입력값과 출력값, 실행 결과를 단계별로 확인할 수 있어 오류 분석과 디버깅이 용이하였다.

반면 Zapier는 단계별 설정 방식으로 구성되어 초보자도 빠르게 자동화를 구축할 수 있었다. 그러나 복잡한 조건 분기가 많아질 경우 전체 흐름을 한눈에 확인하기 어렵고 무료 플랜에서 사용할 수 있는 기능에도 일부 제한이 존재하였다.

따라서 간단한 자동화 업무는 Zapier가 더 빠르고 쉽게 구축할 수 있었으며, 조건 분기와 데이터 처리 과정이 많은 자동화 업무는 Make가 더 적합하다고 판단하였다.

**최종 도구 선택 이유**

본 프로젝트에서는 최종 구현 도구로 Make를 선택하였다. 선정 이유는 다음과 같다.
- Router 기능을 이용하여 조건 분기를 직관적으로 구현할 수 있음
- 데이터 흐름을 시각적으로 확인할 수 있음
- 실행 로그와 오류 원인을 상세하게 분석할 수 있음
- 무료 플랜에서도 충분한 기능 제공
- 교육용 과제로 자동화 구조를 설명하기에 적합함

위와 같은 이유로 본 프로젝트에서는 Make를 최종 자동화 플랫폼으로 선정하였다.

### 11. 장단점 분석

**Make**
- 장점
  - 복잡한 워크플로우 구현 가능

  - 데이터 흐름을 시각적으로 확인 가능

  - 직관적인 인터페이스
  - 빠른 구축 가능
  - 초보자도 쉽게 사용 가능
- 단점

**Make**
- 복잡한 업무 자동화
- 다수의 조건 분기
- 대규모 워크플로우

- 빠른 구축이 필요한 경우
- 비개발자 사용자

### 13. 결론

## 프로젝트 2. 교수용 성적 자동 산출 및 학사경고 자동 알림 시스템

### 1. 프로젝트 개요
본 프로젝트는 교수자의 성적 관리 업무를 자동화하기 위해 Make를 활용하여 구축한 자동화 시스템이다. 학생의 중간고사 점수와 기말고사 점수를 Google Sheets에 입력하면 평균 점수를 자동 계산하고 학점을 자동 부여한다. 이후 학점에 따라 학사경고 대상 여부를 판단하여 이메일을 발송하고 결과를 별도 시트에 자동 저장하도록 구성하였다. 반복적으로 수행하던 성적 계산, 학사경고 판정, 이메일 발송 및 결과 정리 업무를 자동화하여 업무 효율성을 향상시키는 것을 목표로 하였다.

### 2. 자동화 대상 업무

**자동화 전**
- 교수자가 중간고사와 기말고사 점수를 확인
- 평균 점수 직접 계산
- 학점 수동 산출
- 학사경고 여부 확인
- 이메일 개별 발송
- 결과 별도 정리

**자동화 후**
- 평균 자동 계산
- 학점 자동 부여
- 학사경고 여부 자동 판단
- 이메일 자동 발송
- 결과 시트 자동 저장

### 3. 선정 도구 및 선정 이유

**선정 도구**
- Make
- Google Sheets
- Gmail

**선정 이유**

본 프로젝트에서는 Make와 Zapier를 비교한 후 Make를 최종 자동화 도구로 선정하였다.

첫째, Make는 자동화 과정을 시각적인 워크플로우 형태로 제공하여 전체 데이터 흐름을 한눈에 파악할 수 있다. 본 프로젝트에서는 Google Sheets에서 학생 정보를 입력받고, Router를 통해 학사경고 대상 학생과 일반 학생을 분기한 뒤 Gmail 발송 및 Google Sheets 저장까지의 과정을 하나의 화면에서 확인할 수 있었다. 따라서 자동화 구조를 이해하고 설명하기에 용이하였다.

둘째, Make의 Router 기능은 조건 분기 구현에 적합하였다. 본 프로젝트에서는 학점이 F인 학생에게는 학사경고 안내 메일을 발송하고, F가 아닌 학생에게는 성적 결과 안내 메일을 발송해야 했다. Make는 Router를 통해 이러한 조건 분기를 직관적으로 구현할 수 있었으며, 각 경로의 실행 과정을 쉽게 확인할 수 있었다.

셋째, 무료 플랜에서도 다양한 기능을 활용할 수 있다는 장점이 있었다. Zapier의 경우 일부 조건 분기 기능은 유료 플랜에서 제공되지만, Make는 Router 기능을 무료로 사용할 수 있어 별도의 비용 없이 프로젝트를 구현할 수 있었다.

넷째, 실행 결과 확인과 오류 분석이 편리하였다. Make는 각 모듈의 입력 데이터, 출력 데이터, 실행 성공 여부를 확인할 수 있어 자동화 과정의 검증이 용이하였다. 실제 테스트 과정에서 Google Sheets 연결 오류가 발생하였으나 오류 메시지와 실행 로그를 통해 원인을 확인하고 수정할 수 있었다. 또한 Gmail 발송 여부와 Google Sheets 저장 여부를 단계별로 확인할 수 있어 디버깅과 결과 검증에 큰 도움이 되었다.

마지막으로 Make는 교육 목적의 프로젝트에 적합하였다. 본 과제의 목표는 단순히 자동화를 구현하는 것이 아니라 Trigger, Router, Action의 개념을 이해하고 자동화 흐름을 설명하는 것이다. Make는 모듈 간 연결 관계를 시각적으로 제공하여 자동화 구조를 학습하고 설명하기에 적합하다고 판단하였다.

위와 같은 이유로 본 프로젝트에서는 Make를 최종 자동화 도구로 선정하였다.

### 6. 워크플로우 설계

**자동화 흐름**
```
Google Sheets (새 행 추가)
↓
Router
├─ 학점 = F
│   ├─ Gmail
│   │   학사경고 안내 메일
│   └─ Google Sheets
│       학사경고 안내 시트 저장
└─ 학점 ≠ F
    ├─ Gmail
    │   성적 결과 안내 메일
    └─ Google Sheets
        성적결과 시트 저장
```

**Trigger와 Action의 역할**

자동화 시스템은 일반적으로 Trigger와 Action으로 구성된다.

Trigger는 자동화를 시작시키는 이벤트를 의미한다. 사용자의 입력이나 데이터 변경과 같은 특정 조건이 발생하면 자동화가 실행된다. 본 프로젝트에서는 Google Sheets의 새로운 행(Row) 추가가 Trigger 역할을 수행한다. 즉, 학생의 성적 정보가 입력되면 자동화가 시작된다.

Action은 Trigger 이후 수행되는 실제 작업을 의미한다. 이메일 발송, 데이터 저장, 메시지 전송 등이 대표적인 Action이다. 본 프로젝트에서는 Gmail을 이용한 성적 안내 메일 발송과 Google Sheets 결과 시트 저장이 Action 역할을 수행한다.

따라서 본 프로젝트의 전체 흐름은 다음과 같다.

```
Google Sheets (Trigger) → Router (조건 분기) → Gmail (Action) → Google Sheets (Action)
```

즉, 학생 성적 입력이 자동화의 시작점이 되고, 이메일 발송과 결과 저장이 실제 처리 작업을 수행하는 구조로 설계하였다.

**조건 분기 증가 시 구조화 방안**

현재 프로젝트는 학점이 F인 학생과 F가 아닌 학생으로 분기하는 2개의 조건만 사용하였다. 향후 조건이 증가할 경우 Router를 활용하여 분기를 세분화할 수 있다. 예를 들어 다음과 같이 확장할 수 있다.

- A 이상 : 우수 성적 안내 메일
- B : 일반 성적 안내 메일
- C : 학습 권장 메일
- D : 성적 주의 안내 메일
- F : 학사경고 안내 메일

또한 각 분기별 Gmail 모듈과 Google Sheets 저장 모듈을 별도로 구성하여 관리할 수 있다. 이와 같이 기능별로 모듈을 분리하면 유지보수성과 가독성을 높일 수 있으며 향후 기능 확장도 용이하다.

### 7. Make 시나리오 구성

**Step 1. Trigger**
Google Sheets의 Watch New Rows 모듈을 사용하여 새로운 학생 데이터가 입력되면 자동으로 감지하도록 설정하였다.

**Step 2. Router**
학점을 기준으로 두 개의 경로로 분기하도록 설정하였다.

- 경로 1
  - 조건: 학점 = F
  - 실행: 학사경고 메일 발송, 학사경고 안내 시트 저장
- 경로 2
  - 조건: 학점 ≠ F
  - 실행: 성적 결과 메일 발송, 성적결과 시트 저장

**Step 3. Gmail**

학사경고 메일
- 제목: 학사경고 안내
- 본문:
  ```
  안녕하세요. 학사경고 대상자로 분류되었습니다.
  이름 : {{이름}}
  평균 : {{평균}}
  학점 : {{학점}}
  담당 교수와 상담하시기 바랍니다.
  자동 발송 메일입니다.
  ```

성적 결과 메일
- 제목: 성적 결과 안내
- 본문:
  ```
  안녕하세요. 성적 결과를 안내드립니다.
  이름 : {{이름}}
  평균 : {{평균}}
  학점 : {{학점}}
  한 학기 동안 수고 많으셨습니다.
  자동 발송 메일입니다.
  ```

**Step 4. Google Sheets**

학사경고 안내 시트
- 자동 저장 항목: 이름, 평균, 학점, 이메일

성적결과 시트
- 자동 저장 항목: 이름, 평균, 학점, 이메일

### 10. 프로젝트 회고
이번 프로젝트를 통해 Make를 활용한 노코드 자동화 시스템을 직접 설계하고 구현할 수 있었다.

특히 Trigger, Router, Action의 개념을 실제 업무에 적용하면서 반복 업무를 자동화하는 방법을 이해할 수 있었다.

Google Sheets, Gmail, Make를 연동하여 실제 교수자의 성적 관리 업무를 자동화할 수 있었으며, 조건 분기를 활용하여 학사경고 대상 학생과 일반 학생을 구분 처리하는 기능도 구현하였다.

향후에는 Slack 알림, Notion 연동, 생성형 AI를 활용한 성적 분석 기능 등을 추가하여 더욱 확장된 자동화 시스템으로 발전시킬 수 있을 것으로 기대한다.

본 프로젝트를 통해 노코드 자동화 도구를 활용하여 실제 업무 프로세스를 자동화할 수 있음을 확인하였다. 특히 조건 분기(Router)를 활용하여 서로 다른 업무 흐름을 자동 처리할 수 있었으며, 반복 업무 감소 및 업무 효율 향상 효과를 확인할 수 있었다.

### 11. 실패 대응 및 운영 전략
자동화 시스템은 Google Sheets, Gmail 등 외부 서비스와 연동되므로 연결 오류가 발생할 수 있다. 본 프로젝트에서는 Make의 실행 로그를 통해 오류 발생 여부를 확인할 수 있도록 설계하였다. 실제 구현 과정에서도 Google Sheets 연결 오류(ConnectionError)가 발생하였으며, 실행 로그를 통해 오류 원인을 확인하고 연결을 재설정하여 문제를 해결하였다.

운영 환경에서는 다음과 같은 방식으로 안정성을 높일 수 있다.
- Make 실행 로그를 주기적으로 확인하여 오류 발생 여부를 모니터링한다.
- Gmail 발송 실패 시 관리자 이메일로 오류 알림을 전송한다.
- 일정 시간 후 자동 재시도 기능을 설정하여 일시적인 네트워크 오류에 대응한다.
- Gmail 발송 실패 시 Google Sheets에 오류 내역을 저장하여 대체 경로로 활용한다.

이를 통해 자동화 시스템의 안정성과 신뢰성을 향상시킬 수 있다.

### 12. 노코드 자동화의 한계와 확장 아이디어
노코드 자동화는 반복 업무를 빠르게 구현할 수 있다는 장점이 있지만 복잡한 데이터 처리와 고급 비즈니스 로직 구현에는 한계가 있다. 예를 들어 학생의 과거 성적, 출석률, 과제 제출 현황을 종합적으로 분석하여 개인별 학습 전략을 제안하는 기능은 노코드만으로 구현하기 어렵다.

이러한 기능은 Python과 같은 프로그래밍 언어를 활용하여 구현할 수 있다. 향후에는 OpenAI API와 Python을 연동하여 학생별 학습 데이터를 분석하고 맞춤형 학습 피드백을 자동 생성하는 시스템으로 확장할 수 있다. 또한 데이터베이스와 연동하여 대규모 학생 데이터를 처리하는 기능도 추가할 수 있다.
