## B1-3. 코딩 없이 마우스 클릭만으로 일이 자동으로 돌아가게 만들기

### 프로젝트 1. 자동화 도구 비교 구현 (Make vs Zapier)

동일한 업무 자동화 워크플로우를 Make와 Zapier 두 가지 노코드 자동화 플랫폼으로 각각 구현하고 비교 분석한 프로젝트입니다.

- **자동화 대상 업무**: 교수자의 학생 성적 관리 업무
- **워크플로우**: Google Sheets에 학생 성적 입력 → 학점 확인 → 조건 분기(학점 = F / 학점 ≠ F) → 학사경고 메일 또는 성적 결과 메일 발송 → 결과를 별도 시트에 저장

#### Make 구현
- 사용 모듈: Google Sheets - Watch New Rows, Router, Gmail - Send Email, Google Sheets - Add a Row
- Router를 통해 학사경고 대상(학점 = F)과 일반 학생(학점 ≠ F)으로 분기

#### Zapier 구현
- 사용 모듈: Google Sheets - New Spreadsheet Row, Paths by Zapier, Gmail - Send Email, Google Sheets - Create Spreadsheet Row
- Paths를 통해 동일한 조건으로 분기

#### 테스트 결과
| 이름 | 평균 | 학점 | 결과 |
| --- | --- | --- | --- |
| 김철수 | 45 | F | 학사경고 안내 메일 발송 성공, 학사경고 안내 시트 저장 성공 |
| 이찬원 | 95 | A+ | 성적 결과 안내 메일 발송 성공, 성적결과 시트 저장 성공 |

#### Make vs Zapier 비교

| 비교 항목 | Make | Zapier |
| --- | --- | --- |
| UI/UX | 시각적 노드 기반 | 단계형 리스트 기반 |
| 설정 난이도 | 다소 높음 | 비교적 쉬움 |
| 조건 분기 | Router 사용, 상세 확인 가능 | Paths 사용, 간단한 실행 기록 제공 |
| 확장성 | 높음 | 보통 |
| 학습 난이도 | 중간 | 낮음 |
| 초보자 친화성 | 보통 | 높음 |
| 복잡한 자동화 | 강점 | 제한적 |
| 무료 플랜 | 월 1,000 Operations | 월 100 Tasks 수준 |
| AI 연동 | 다양한 AI 모듈 지원 | AI Action 지원 |

**결론**: 간단한 자동화에는 Zapier가 적합하고, 복잡한 자동화 및 다수의 조건 분기가 필요한 경우 Make가 더 적합하다.

### 프로젝트 2. 교수용 성적 자동 산출 및 학사경고 자동 알림 시스템

Make를 활용하여 학생의 중간고사·기말고사 점수를 입력하면 평균과 학점을 자동 산출하고, 학사경고 대상 여부를 판단하여 이메일을 발송하고 결과를 시트에 자동 저장하는 시스템을 구축한 프로젝트입니다.

#### 선정 도구
- **Make** (자동화 워크플로우)
- **Google Sheets** (데이터 입력 및 결과 저장)
- **Gmail** (이메일 발송)

#### Make 선정 이유
1. 시각적 워크플로우로 전체 데이터 흐름을 한눈에 파악 가능
2. Router 기능으로 조건 분기 구현에 적합
3. 무료 플랜에서도 Router 등 다양한 기능 활용 가능
4. 실행 결과 및 오류 분석이 용이 (모듈별 입력/출력/성공 여부 확인)
5. Trigger, Router, Action 개념을 학습하고 설명하기에 적합

#### 성적 산출 기준

| 평균 점수 | 학점 |
| --- | --- |
| 95 이상 | A+ |
| 90 이상 | A |
| 85 이상 | B+ |
| 80 이상 | B |
| 75 이상 | C+ |
| 70 이상 | C |
| 65 이상 | D+ |
| 60 이상 | D |
| 60 미만 | F |

#### Google Sheets 구성
- **성적입력 시트**: 이름, 중간고사, 기말고사, 평균, 학점, 이메일
- **학사경고 안내 시트**: 학사경고 대상 학생 정보 자동 저장
- **성적결과 시트**: 일반 학생 성적 정보 자동 저장

#### 워크플로우
```
Google Sheets (새 행 추가)
↓
Router
├─ 학점 = F → Gmail(학사경고 안내 메일) → 학사경고 안내 시트 저장
└─ 학점 ≠ F → Gmail(성적 결과 안내 메일) → 성적결과 시트 저장
```

#### Make 시나리오 구성
- **Step 1. Trigger**: Google Sheets - Watch New Rows로 새 학생 데이터 감지
- **Step 2. Router**: 학점 기준 두 경로로 분기 (학점 = F / 학점 ≠ F)
- **Step 3. Gmail**: 학사경고 안내 메일 / 성적 결과 안내 메일 발송 (이름, 평균, 학점을 동적으로 매핑)
- **Step 4. Google Sheets**: 학사경고 안내 시트 / 성적결과 시트에 이름, 평균, 학점, 이메일 자동 저장

#### 테스트 결과

| 이름 | 평균 | 학점 | 결과 |
| --- | --- | --- | --- |
| 김철수 | 45 | F | 학사경고 안내 메일 발송 성공, 학사경고 안내 시트 저장 성공 |
| 이찬원 | 95 | A+ | 성적 결과 안내 메일 발송 성공, 성적결과 시트 저장 성공 |

#### 프로젝트 회고
Make를 활용한 노코드 자동화 시스템을 직접 설계하고 구현하면서 Trigger, Router, Action 개념을 실제 업무에 적용해 반복 업무를 자동화하는 방법을 이해할 수 있었다. Google Sheets, Gmail, Make를 연동하여 교수자의 성적 관리 업무를 자동화하고, 조건 분기를 통해 학사경고 대상 학생과 일반 학생을 구분 처리하는 기능을 구현하였다. 향후에는 Slack 알림, Notion 연동, 생성형 AI를 활용한 성적 분석 기능 등을 추가하여 자동화 시스템을 확장할 수 있을 것으로 기대한다.
